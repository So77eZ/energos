# Замечания и предложения по проекту Energos

## Критичные проблемы

### 1. ~~CORS + credentials~~ ✅ Исправлено (`backend/main.py`)

`allow_origins=["*"]` вместе с `allow_credentials=True` — нарушение стандарта CORS. Браузер запрещает это сочетание, и либо credentials не работают, либо сервер уязвим к CSRF.

**Что сделано:** список разрешённых origin'ов вынесен в переменную окружения `ALLOWED_ORIGINS`. По умолчанию разрешены только `http://localhost:3000` и `http://127.0.0.1:3000`. В продакшне задать: `ALLOWED_ORIGINS=https://yourdomain.com`.

### 2. Нет проверки владельца отзыва на бэкенде (`backend/app/api/endpoints/reviews.py:52–60`)

`PUT /reviews/{id}` не проверяет `review.user_id == current_user.id` — любой авторизованный пользователь может изменить чужой отзыв.

**Исправление:** добавить проверку перед обновлением, возвращать 403 если не владелец и не admin.

### 3. ~~Фронтенд не проверяет роль пользователя~~ ✅ Исправлено

Реализована серверная проверка роли на всех admin-страницах и страницах отзывов (коммит `90144ae`).

### 4. Нет валидации пароля при регистрации (`backend/app/api/models/auth.py`)

`UserCreate` не имеет ограничений на длину и сложность пароля — можно зарегистрироваться с пустым или однобуквенным паролем.

**Исправление:** добавить `Field(min_length=8)` и при необходимости regex-валидатор.

## Средние замечания

### 5. Нет ForeignKey в моделях (`backend/app/api/models/reviews.py:13–15`)

`energy_drink_id` и `user_id` объявлены просто как `int` без SQLAlchemy `ForeignKey`. Возможны «висячие» отзывы после удаления напитка или пользователя.

**Исправление:** добавить `ForeignKey("energy_drinks.id")` и `ForeignKey("users.id")`.

### 6. Загрузка изображений без валидации типа файла (`backend/app/database.py`)

Supabase Storage принимает любой Content-Type без ограничений.

**Исправление:** разрешить только `image/jpeg`, `image/png`, `image/webp`; ограничить размер (например, 5 МБ).

### 7. Нет пагинации

Все напитки и отзывы загружаются одним запросом. При росте базы данных это создаст проблемы с производительностью.

**Исправление:** добавить параметры `limit` и `offset` в API, реализовать постраничную навигацию на фронтенде.

### 8. Дублирование bearer-заголовка в API-клиентах

В `DrinkApi`, `ReviewApi`, `AuthApi` одинаковая конструкция `Authorization: Bearer <token>`.

**Исправление:** вынести в одну утилиту (например, `bearerHeaders(token)`).
